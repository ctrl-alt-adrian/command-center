Passing resume context to search suggestion logic produces fundamentally better results than query-only suggestions.

RoleNext search suggestions based on query alone were generic. Adding resume context (skills, job history, titles) transformed suggestion quality without changing the suggestion algorithm—same input type, dramatically different output. The resume grounds suggestions in the user's actual background, preventing irrelevant suggestions and making recommendations contextual to their experience. This works because it's free—no algorithm changes needed, just threading existing context through the frontend and backend.
