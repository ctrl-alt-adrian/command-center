If a feature requires users to discover it through a tab or hidden action, most won't find it. Make guidance visible by default; use interaction for navigation, not revelation.

During UX review of search intent guidance, the user stated: if intent help is only enabled by tabbing, most users will never know it exists. That is the wrong UX. The principle is universal: visible-by-default triggers based on user context (keystroke, debounce, form state) ensure adoption without discovery work. Tabs and modals are for navigation and keyboard accessibility, not for gatekeeping features. When a feature is hidden, it becomes invisible, regardless of its utility.
