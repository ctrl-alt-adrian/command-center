Condense large external style guides into 20-30 actionable one-liners per language, keeping full docs as reference material.

RoleNext consumed a 400-line Go guide, 70-rule Vercel React best practices, and the PostgreSQL anti-patterns wiki. Full documents bloat context. Distilled each into ~20-30 core rules as one-liners (no examples, no verbose explanations) fitting into 40-line rule files. Full docs stay at source as reference. This trades depth for signal-to-noise ratio when context budget is finite.
