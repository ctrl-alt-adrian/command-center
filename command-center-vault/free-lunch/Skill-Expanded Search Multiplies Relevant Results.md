Extract atomic skills from user input and run parallel searches instead of one keyword.

Single-keyword job search misses relevant opportunities. RoleNext extracts the user's top skills from their resume (S1 output) and runs parallel JSearch queries for each skill, then deduplicates and blends results. This catches jobs where the title doesn't match but the skill set does. For example, a designer with Figma and usability research skills finds UX roles, product roles, and design-adjacent research roles instead of just designer listings. The pattern: expand your search space first, then filter on client side.
