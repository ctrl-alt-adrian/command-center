PDFs that extract to only whitespace should be rejected at upload time with specific user feedback, not silently accepted and later cause confusing failures downstream.

During rolenext pre-launch work, a user uploaded a PDF generated by the app itself, but it only contained blank pages. The upload handler accepted it (no validation), then later operations failed with generic 'no resume provided' errors. The fix: after extracting text, check that the result contains non-whitespace. Reject with a user-facing error pointing to image-only or scan-only PDFs as the likely cause. This prevents silent garbage-in-garbage-out situations. Applied to both the upload handler and the direct-analyze endpoint.
