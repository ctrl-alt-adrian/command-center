Search quality validation must account for user context. Generic validation rules fail when you don't know who is searching.

RoleNext's search system learned that quality judgments differ significantly based on the user's job history. A skill suggestion appropriate for a junior engineer is hallucination for a senior one. This pushed personalization earlier in the validation pipeline: instead of generic intent checks, validation now uses resume data to contextualize what makes a search high-quality. The lesson generalizes: systems claiming to understand quality without understanding the user will miss context-dependent mistakes.
