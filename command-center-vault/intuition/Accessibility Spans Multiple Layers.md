Accessibility requires independent, simultaneous attention to contrast ratios, semantic HTML, cursor feedback, and visual affordances.

In RoleNext's settings page redesign, the team discovered that accessibility is not a single concern. You must attend to: color contrast ratios (handled by the token system), semantic HTML (required fields marked with asterisks and proper labels), cursor feedback (disabled buttons use not-allowed cursor), and visual affordances (generic User icon instead of initials for missing data). Fixing one layer does not guarantee accessibility; all must be in place.
