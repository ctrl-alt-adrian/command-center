When a feature moves from planned to shipped, update the handoff doc to summarize what was delivered instead of keeping the stale planning spec.

Handoff docs often contain detailed 'What's Next' planning specs. When that work ships, keeping the spec as-is leaves stale documentation that doesn't reflect current state. Instead, replace it with a concise summary of what was delivered: file list, PR, key changes. Preserve detailed specs in a completed-plans archive for architectural reference. This keeps handoff docs accurate while maintaining documentation history. (RoleNext explainable-scoring, PR #179)
