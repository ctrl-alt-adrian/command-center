Before committing to a design-first workflow, audit the next mock for technical feasibility to catch showstoppers early.

After shipping the dashboard PR, the user ran a feasibility audit on the next Claude Design mock (Search page with live per-source status panel) before locking the workflow for the rest of the app. This caught potential complexity around real-time status updates and allowed scope negotiation before committing implementation resources. The working agreement: designs go through Claude Design first, but implementation only starts after you sign off on feasibility.
