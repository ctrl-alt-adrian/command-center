Keeping docs fresh means removing outdated sections, not just adding new ones.

CLAUDE.md accumulated outdated content from a prior design phase: Pencil mockup workflows, 10 task gates, and verbose specifications no longer in use. Rather than leave them as legacy cruft, the team removed them entirely and replaced with a lean Code Quality section. Docs that don't reflect current practice rot faster than docs that are trimmed ruthlessly. Active removal is part of maintenance.
