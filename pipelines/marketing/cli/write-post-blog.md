# Blog

Write a blog post.

Rules:
- 600-1200 words
- First line is a `# Title` markdown H1
- Open with a specific problem, moment, or realization, not a generic intro
- Use H2 subheadings for sections, fenced code blocks for code, bullet or numbered lists where they read naturally
- Concrete detail over abstraction: real errors, real numbers, real decisions
- End with a short natural takeaway, not a "what do you think?" call-to-action
- For technical posts: structure around problem → investigation → fix → outcome, with code snippets where helpful
- For marketing posts: tell the story behind a decision, show the trade-offs, and include real user feedback or data if present
