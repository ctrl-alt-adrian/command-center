Use a persistent floating action button for feedback collection instead of burying it in a dropdown menu, trading real estate for discoverability.

RoleNext initially offered feedback and bug reporting from the user dropdown menu. User feedback revealed redundancy and poor discoverability. A floating action button (FAB) makes the feature always visible without interrupting the main interface. The tradeoff: the FAB consumes screen real estate but appears only when needed (collapsed to an icon, expands on hover). The dropdown was cleaned up to focus on navigational items (Settings, Log out) instead of feature entry points. The FAB also avoids the cognitive load of nested menus.
